## 🛠️ Tecnologias, Ecossistemas e Abordagens

### 🔴 Oracle SQL Developer (PL-SQL & Consultas Corporativas)
Uso focado na ferramenta corporativa **SQL Developer** para conectar a instâncias robustas do motor Oracle. 
* **Abordagem e Prática:**
  * Escrita de instruções SQL complexas com sintaxe nativa Oracle.
  * Criação e manipulação de objetos de banco de dados (Tabelas, Views, Sequências e Índices).
  * Criação de scripts DDL estruturados para automação de ambientes corporativos.
  * Resolução de exercícios complexos simulando cenários reais de grandes empresas que utilizam a arquitetura Oracle.

### 🔵 Microsoft SQL Server (T-SQL & Automação de Negócios)
Estudos dedicados à manipulação e engenharia de dados usando a linguagem procedural estendida da Microsoft (Transact-SQL).
* **Abordagem e Prática:**
  * Desenvolvimento de consultas inteligentes estruturando junções complexas de dados (`JOINs`).
  * Manipulação fina de dados em massa usando subconsultas, agrupamentos (`GROUP BY`) e filtros avançados (`HAVING`).
  * Implementação de restrições de integridade (`Constraints`) como chaves primárias, estrangeiras e validações customizadas (`CHECK`).

### 🐬 MySQL (Modelagem Relacional & Desenvolvimento Web)
Foco em ambientes ágeis e integrações com aplicações Full-Stack (como Node.js, Prisma ORM e ecossistemas web).
* **Abordagem e Prática:**
  * Criação de diagramas Entidade-Relacionamento (DER) e transformação de modelos conceituais em esquemas físicos estáveis.
  * Scripts práticos de criação, leitura, atualização e exclusão de dados (Operações CRUD complexas via DML).
  * Otimização de bancos de dados locais para conectar com APIs em JavaScript/TypeScript.

## 🚀 Conceitos Técnicos Consolidados neste Repositório

* **Linguagem DDL (Data Definition Language):** Domínio completo dos comandos `CREATE`, `ALTER` e `DROP` para a engenharia e arquitetura estrutural das tabelas.
* **Linguagem DML (Data Manipulation Language):** Escrita assertiva de comandos `INSERT`, `UPDATE` e `DELETE` gerenciando a integridade dos registros.
* **Consultas Avançadas (DQL):** Uso inteligente de funções agregadoras (`SUM`, `COUNT`, `AVG`, `MAX`, `MIN`), relacionamentos (`INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`) e subconsultas aninhadas.
* **Integridade Referencial:** Aplicação rigorosa de regras de negócio através de chaves primárias (`PRIMARY KEY`) e chaves estrangeiras (`FOREIGN KEY`) para evitar anomalias de dados.
